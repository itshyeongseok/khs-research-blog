Learning to Walk in Minutes Using Massively Parallel Deep Reinforcement Learning
Tier 1 — 최상위
Nikita Rudin, David Hoeller, Philipp Reist, Marco Hutter
Proceedings of the 5th Conference on Robot Learning (CoRL 2021), PMLR vol. 164, pp. 91-100, 2022 (arXiv:2109.11978). Open-source code: legged_gym (https://leggedrobotics.github.io/legged_gym/).
논문 요약
본 논문은 단일 워크스테이션 GPU의 대규모 병렬성을 활용하여 실제 다리형 보행 정책을 빠르게 생성하는 학습 환경을 제시한다. NVIDIA의 Isaac Gym을 사용해 시뮬레이션과 정책 업데이트를 모두 GPU에서 종단간(end-to-end)으로 수행한다. 저자들은 표준적인 온-폴리시(on-policy) RL 정식화(GAE를 사용하는 맞춤형 PPO)와 그 하이퍼파라미터가, 수천 대의 로봇을 로봇당 매우 적은 스텝으로 시뮬레이션하는 고도 병렬 환경에 어떻게 적응되어야 하는지를 분석하며, 각 로봇의 성능에 맞추어 지형 난이도를 거의 추가 비용 없이 조정하는 새로운 게임 기반 자동 커리큘럼을 도입한다. ANYmal 4족 로봇을 평지에서 보행하도록 학습시키는 데 4분 미만, 불규칙한 지형에서는 약 20분이 걸려, 기존 연구 대비 여러 자릿수에 달하는 속도 향상을 보인다. 학습된 정책은 단순한 고유수용성(proprioceptive) 관측값과 지형 높이 스캔을 사용하고, 보행 방식에 특화된 보상 없이 단순한 보상 항만을 사용하며, 실제 ANYmal 로봇뿐 아니라 ANYmal B/C-팔 장착형, Unitree A1, 2족 로봇 Cassie로 전이되어 접근법의 일반성을 검증한다. 학습 코드는 학습형 다리 보행 연구를 가속하기 위해 legged_gym으로 오픈소스화되었다.

Introduction — 연구 동기 (클릭하여 자세히)
서론은 심층 RL을 로봇공학을 위한 강력하지만 데이터 소모적인 도구로 규정하는데, 현재의 시뮬레이터로는 학습 시간이 며칠, 몇 주, 몇 달로 늘어나 하이퍼파라미터 튜닝도 느려진다고 본다. 저자들은 단일 GPU에서 온-폴리시 심층 RL을 위해 대규모 병렬성을 활용할 것을 제안하며, 표준 RL 정식화와 하이퍼파라미터가 어떻게 적응되어야 하는지를 검토하고 게임 기반 자동 커리큘럼을 도입한다. 그들은 이전에 수십~수백 시간이 필요했던 지각형(perceptive) 다리 보행 방법들과 대비하여, 단일 GPU에서 20분 미만에 지각형 정책을 학습하고 이를 하드웨어에서 검증할 수 있음을 보인다.
Related Works — 관련 연구 (클릭하여 자세히)
관련 연구는 서론과 방법 전반에 걸쳐 두 축, 즉 시뮬레이션/학습 처리량과 학습형 다리 보행을 따라 엮여 있다. 처리량 측면에서 저자들은 분산 CPU 클러스터의 그래디언트 평균화, 병렬 시뮬레이션과 다중 GPU 학습의 결합, 대규모 배치의 시각 내비게이션 시뮬레이션, Humanoid 과제용 GPU 가속 물리, 그리고 미분가능 TPU 시뮬레이터(Brax)를 인용한다. 보행 측면에서는 복잡한 모델 기반 지각 제어기와 떠오르는 학습 기반 방법(맹목적 강건 정책, 학습+최적제어 하이브리드, 완전 학습형 교사-학생 접근법)을 대비하며, 자신들의 빠른 단일 GPU 파이프라인을 GPU 시뮬레이션 연구의 연장선상에 두면서 도전적인 실제 과제에 적용한다.
Method — 방법론 (클릭하여 자세히)
방법론은 대규모 병렬 온-폴리시 RL을 중심으로 한다: 맞춤형 PPO 구현이 모든 데이터와 연산을 GPU에 유지하고, Isaac Gym의 PhysX 엔진이 수천 대의 로봇을 시뮬레이션하며 그 파이프라인 연산이 벡터화된다. 핵심 하이퍼파라미터 통찰은 배치 크기 B = n_robots * n_steps가 합리적인 수준을 유지해야 한다는 것으로, n_robots를 여러 자릿수만큼 늘리면 n_steps를 작게 강제하게 된다(다만 GAE가 다중 스텝 보상을 필요로 하므로 약 25스텝 미만으로는 내릴 수 없다). 또한 비정상적으로 큰 미니배치를 함께 사용한다. 리셋은 예측 가능한 실패/목표 리셋과 예측 불가능한 타임아웃으로 나뉘며, 후자는 크리틱 타깃을 자신의 가치 예측으로 부트스트랩하여 처리한다. 게임 기반 커리큘럼은 각 로봇에 지형 유형과 난이도 레벨을 부여하고, 로봇이 얼마나 멀리 걷는지에 따라 올리거나 내리며, 최고 난이도를 통달한 로봇은 망각을 피하기 위해 무작위 레벨로 되돌린다.
Experiment — 실험 (클릭하여 자세히)
실험은 먼저 병렬 로봇 수가 최종 정책 성능과 학습 시간에 미치는 영향을, 고정된 배치 크기에서 128부터 16384까지 스윕하며 연구한다. 로봇이 너무 많으면 성능이 붕괴하고(로봇당 지평이 너무 짧아짐), 너무 적으면 샘플 다양성이 줄어든다; 배치 크기 약 100k~200k에서 2048~4096 로봇이 최적의 절충을 주며, 학습 시간은 약 4000 로봇까지 거의 선형으로 확장된다. 4096 로봇, 배치 98304, 1500 업데이트(20분 미만)로 학습한 배치 정책은 0.2 m 높이 계단까지 거의 100% 성공률을 달성하며, 장애물과 경사에 대한 별도 시험도 수행한다. 일반성은 동일 설정을 ANYmal B, 팔 장착 ANYmal C, Unitree A1, 2족 Cassie에 재학습하여 입증한다.
Discussion — 논의 (클릭하여 자세히)
논의는 병렬 환경의 절충과 sim-to-real 전이를 해석한다. 로봇 수에는 최적 구간이 있다: 샘플 다양성과 짧은 벽시계 시간을 위해 충분하되, 로봇당 지평이 학습을 붕괴시킬 만큼 많지는 않아야 한다; 배치 크기를 늘리면 성능은 돕지만 학습 시간도 길어지므로 균형이 필요하다. 하드웨어에서 고정된 정책은 센서 관측값을 필터링 없이 목표 관절 위치로 직접 매핑하지만, 불완전한 Lidar 구축 높이 지도가 강건성을 떨어뜨려 배치 속도를 0.6 m/s로 제한한다. 저자들은 더 큰 강건성으로 가는 경로로 교사-학생 결합을 제시한다. 부록 분석은 타임아웃 부트스트래핑이 전체 보상을 약 10~20% 개선하고 크리틱 손실을 낮춤을 보인다.
Conclusion — 결론 (클릭하여 자세히)
저자들은 수천 대의 병렬 로봇을 사용하는 종단간 GPU 파이프라인과 게임 기반 커리큘럼을 통해 복잡한 실제 로봇 과제를 온-폴리시 DRL 알고리즘으로 분 단위에 학습할 수 있으며, 학습 시간을 여러 자릿수만큼 줄인다고 결론짓는다. 그들은 특정 보행을 장려하지 않는 단순한 관측/행동 공간과 직관적인 보상으로 과제가 해결됨을, 그리고 목표가 최대 강건성이 아니라 하드웨어에서 사용 가능한 정책의 기록적인 빠른 학습임을 강조한다. 그들은 필요한 학습 시간에 대한 연구자들의 기대를 바꾸기를 바라며, 다른 많은 과제도 대규모 병렬 환경의 혜택을 받을 것으로 기대하고, 분야를 가속하기 위해 legged_gym을 공개한다.
